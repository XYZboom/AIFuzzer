### 一、关键 Bug 发现

#### 🔴 **致命 Bug: ShapeAdapter.adapt() 中形状映射错误**

**位置**: `ShapeAdapter.kt` 第 35-36 行

```kotlin
for ((output, shape) in node.outputs.zip(shapeMap.values.toList().takeLast(node.outputs.size))) {
    output.type.shape = shape
}
```

**问题**:
- `shapeMap` 是 `Map<String, UirShape>`，包含**所有节点所有输出**的形状（valueId → Shape）
- `shapeMap.values.toList().takeLast(node.outputs.size)` 取的是 **整个 Map 的最后 N 个值**，而不是当前节点输出的形状
- 这会导致形状**错配**：例如节点 A 的输出被赋予节点 B 的形状

**正确实现**:
```kotlin
for (output in node.outputs) {
    val shape = shapeMap[output.valueId]
        ?: throw IllegalStateException("Shape not found for output ${output.valueId} in node ${node.name}")
    output.type.shape = shape
}
```

这是导致 `ConcurrentModificationException` 的**根本原因**之一，因为 `shapeMap` 在 `inferGraphShapes` 中被构建后，这里的错误映射可能导致后续访问时状态不一致。

---

#### 🔴 **ConcurrentModificationException 根因**

实际运行时 **100% 失败**，全部抛出 `ConcurrentModificationException`。经过追踪：

**问题链**:
1. `UirGenerator.generate()` → `logicGen.generate()` → `buildGraph { ... }`
2. `buildGraph` 中 `nodes.forEach { nodes.add(it) }` — 注意局部变量 `nodes` 和 builder 属性 `this.nodes` 同名
3. Builder 最终将 **同一个 MutableList** 传给 `UirGraphImpl`
4. `ShapeAdapter.adapt()` 迭代 `graph.nodes` 并修改 `output.type.shape`
5. 如果某个内部操作（如 `inferGraphShapes` 或后续的 `UirSerializer.toJsonl`）**再次迭代同一个列表**，就会触发并发修改异常

**验证**:
```bash
./gradlew run
# 每个测试都抛出 ConcurrentModificationException
```

### 五、建议的修复方案

```kotlin
// ShapeAdapter.kt - 修复形状映射
for (node in graph.nodes) {
    for (output in node.outputs) {
        val shape = shapeMap[output.valueId]
            ?: throw IllegalStateException("Shape not found for output ${output.valueId}")
        output.type.shape = shape
    }
}

// UirSerializer.kt - 添加 metadata 序列化
// 在 visitMetadata 中添加:
put("metadata", buildJsonObject {
    program.metadata.forEach { (k, v) -> put(k, v) }
})

// 反序列化时重建 metadata
// 在 fromJsonl 的 visitMetadata 分支:
val metadata = (obj["metadata"]?.jsonObject)?.let { metaObj ->
    metaObj.entries.associate { (k, v) -> 
        k to (v.jsonPrimitive.contentOrNull ?: v.toString())
    }
} ?: emptyMap()
// 然后在 buildProgram 中设置 metadata
```
