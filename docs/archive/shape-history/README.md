## 经验教训

### 1. 不要过度设计

**错误**：把工程问题抽象成数学问题
**正确**：用最简单的工程方案解决

### 2. 充分测试再宣称完成

**错误**：编译通过就以为没问题
**正确**：运行 `./gradlew run` 验证实际效果

### 3. 诚实面对失败

**错误**：隐瞒问题，宣称"搞定"
**正确**：承认"没搞定"，重新设计

### 4. 听取用户意见

**关键反馈**：形状可以通过固定的操作改变，不需要约束求解
**结果**：方案从复杂到简单，从失败到成功
